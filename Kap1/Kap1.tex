\chapter*{Introducción}
El presente proyecto maneja una perspectiva diferente de las desarrolladas hasta la época en el ámbito del procesamiento de datos Rinex, es un planteamiento que se ha venido dando en los últimos años con la aparición del termino y las ideas de la web 2.0 tambien llamada web participativa. En esta se plantea que el conocimiento se puede poner en la web y diferentes actores pueden hacer uso de este. De esta idea de  la web deriva la importancia que tiene este proyecto, al constituirse como el primero que presenta a los usuarios de habla hispana que permite este tipo de procesamiento de datos.\\

En otros países ya se han comenzado a desarrollar varios proyectos de este tipo, algunos ya se encuentran en un estado de madures que parece imposible dado su corto tiempo de desarrollo, pero este hecho casi siempre esta dado por grandes inversiones de capital o el entusiasmo de la comunidad que lo desarrolla. También esta el aspecto teórico, en el cual se plantea usar una técnica que ofrece personalización avanzada del procesamiento de los datos, lo cual hasta el momento solo se ha usado para investigación pero se pretende cambiar ese paradigma.\\

El presente proyecto permite proporcionar a los usuarios que no tienen la posibilidad de adquirir una aplicación avanzada o comercial para procesamiento de datos satelitales, quienes no tienen el tiempo o la disposición para aprender una libre que puede ser algo mas compleja y finalmente a quienes no tienen estaciones de trabajo lo suficientemente poderosas para hacer este tipo de procesamiento.\\

Se pretende que el desarrollo de esta aplicación llegue a todas aquellas personas que requieren usar procesamiento de datos satelitales\cite{HOYER2008}, pero también a las industrias que quieran apoyar la iniciativa en el futuro y finalmente a las comunidades que deseen desarrollar nuevas funcionalidades y proporcionar soporte. La gran limitación  que se presenta es la oposición que pueden llegar a generar las viejas tecnologías al desarrollo de esta.\\

La implementación de una metodología ágil de desarrollo de software que permita avanzar de la mejor manera posible, cumpliendo con los tiempos que se plantean desde el principio. Por parte de las pruebas la metodología es comparativa dado que se plantea usar unos datos de prueba, ejecutar los mismos procesos de esta aplicación en otras que existen en el mercado.
